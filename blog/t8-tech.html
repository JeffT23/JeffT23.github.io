What Exactly is Ruby on Rails?

It's essentially a software library that builds on the Ruby programming language. Rails is a package library (a Ruby Gem), that is installed using the operating system's command-line interface. It serves as a framework for building websites; it combines Ruby with elements of JavaScript, HTML and CSS to create a web application that runs on a web server. Because it runs on a web server, Rails is considered a server-side, or “back end,” web application development platform (as opposed to a web browser which is the “front end”).

Rails has two primary guiding principles. They are as follows:

  • Convention over Configuration (CoC): As opposed to other software systems that require extensive configuration files and settings, Rails relies on assumptions of default values without user intervention. Thus, the developer would only need to specify unconventional aspects of the application. This leads to less code and less repetition.
  • Don't Repeat Yourself (DRY) principle: espouses the need to avoid duplicaton. As processes repeat, a program becomes more harder to maintain and more difficult to debug. Therefore, information should be stored in a single, unambiguous place, and that code should be reused as needed.

    • Rails: The Advantages

    • Programming is easier and faster in Ruby than with competing frameworks. This is due to the "opinionated" nature of the Ruby framework, which provides more productive techniques if you follow the "Rails Way". There's also a vast library of open source code and plugins that further helps to facilitate matters.
    • Rails has a strong testing framework in place.
    • It's open-source, so there are no licensing costs to worry about.

    Rails: The Disadvantages

    • Performance isn't as fast as some competitors such as C or Java. These speed and scalability concerns led Twitter to switch from Rails to Scala (powered by Java).
    • Though this is changing, Rails is not as popular at the moment. Java and PHP are more widely used and have more developers.
    • Some web hosts are unable to use Rails because it uses up more resources than its competitors.
