Making NFL Player Cards with Ruby Classes

January 1, 2014

Growing up, one of my favorite things to do was play the NFL's Madden video game series. Over the years, the video game has gotten so realistic. Not only have the graphics gotten better, but the game's creators have managed to create a ratings system that ranks every player based on their personal attributes. These rankings have grown in popularity, as fans and the players themselves like to debate the numbers assigned to the avatars in the game. Below is a Ruby class structure that will output a player card containing a subset of these rankings. Let's take a look at the code first, and then 'll explain what's going on.

As you know, almost everything in Ruby is an object. A class is simply a way of organizing and producing objects with similar attributes and methods. Here we are grouping together NFL players. We start a class with the "class" keyword followed by the name of the class in CamelCase (here: NflPlayerCard). Next up is the #initialize method which is used by the class to "boot up" each object the class creates. In our example, you can see that each instance of the NFLPlayerCard class is initialized with the player's name, number, and team name. Within the body of the initialize method, we assign instance variables (denoted by an @ symbol in front of the variable name) for these 3 inputs. In our example for instance, we create a troy instance of the NFLPlayerCard class. We initialize this class with "Troy Polamalu", 43, and "Pittsburgh Steelers", respectively. These @name, @number, and @team variables belong to the "troy" instance of NFLPlayerCard, hence the name instance variable. Now, there are other attributes that we want available to us in our NFLPlayerCard class. For these, Ruby has provided us with its attr methods. As you can see, the attr_reader method puts at our disposal the following attributes that we can assign to our player instances: :position, :height, :weight, :awareness, :speed, :strength, :tackling, :break_tackles, :accuracy.
