Arrays & Hashes - NFL Edition

December 7, 2014

One of the cool advantages of using Ruby as a newbie programmer is that it contains a bunch of collections to help you sort and search data. Of these, arrays and hashes are probably the most popular: I’m going to outline the two of them in this blog post. Arrays are ordered, integer-indexed collections of any object. Array indexing starts at 0, and increases from left to right. Or one can work right to left with a negative index, with -1 indicating the last element in the array, and growing more negative as we work our way to the front. Being that today is Sunday, here is an array example for you football fans. I’ve created a new array in the IRB screen that lists the best NFL franchises in terms of number of Super Bowl rings. BestNFLFranchises = ["Steelers", "Cowboys", "49ers", "Packers", "Giants", “Patriots”] irb(main):004:0> BestNFLFranchises[0] => "Steelers" irb(main):005:0> BestNFLFranchises[-6] => “Steelers"

As you can see, the greatest franchise (without a doubt, the Steelers) can be referenced with the 0 index, left to right, or -6 going the other way, both denoting the first item on the list. Let’s move on to hashes. Hashes are a collection type revolving around key value pairs. Indexing isn’t based on an integer value, but rather on the order in which the keys are typed in. Continuing with our Super Bowls example, we can list the key of team name and assign a corresponding value that ties to the number of rings, like so: BestNFLFranchises= {"Steelers"=> 6, "Cowboys"=>5, "49ers"=>5 , "Packers"=> 4, "Giants" => 4, "Patriots"=> 3} We can then call upon a certain franchise by name and the program should spit out the corresponding value, like so: irb(main):002:0> BestNFLFranchises["Steelers"] => 6

Now both arrays and hashes have a slew of useful methods that can be called upon to sort and search and otherwise manipulate data (Ruby-Doc is a great source for finding this information). The main difference seems to be in the way they are indexed. Arrays are limited to integer indexes as noted above, whereas hashes can use any data object as a key (aka index).
